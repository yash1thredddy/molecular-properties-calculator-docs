THE ANALYSES

Batch processing

Process hundreds or thousands of molecules at once from a spreadsheet — upload a column of structures, pick what you want calculated, and download an enriched file with every result filled in.

What batch processing is for

The Single Molecule workspace is perfect for one compound at a time, but real projects come in libraries — a screening plate, a virtual collection, a synthesis series. Batch Processing is the mode that takes a whole spreadsheet of molecules and calculates the same set of numbers for every row in one pass.

You hand it a file with a column of chemical structures (one molecule per row). It reads each structure, computes whatever properties and efficiency indices you asked for, and gives you back your original spreadsheet with a set of brand-new columns appended — ready to open in Excel or feed into your next tool. Nothing about your original data is thrown away; the new numbers simply sit alongside it.

Below is the shape of the journey — your file goes in on the left, gets enriched in the middle, and comes back out on the right with extra columns.

The four-step workflow

From spreadsheet to enriched results in four moves. Each step is explained in full below.

Prepare your file

Make a CSV (a plain-text "comma-separated values" file) or an Excel workbook (.xlsx) with one column holding your chemical structures — one molecule per row. The structures are usually SMILES (a short text recipe for a molecule, e.g. aspirin is CC(=O)OC1=CC=CC=C1C(=O)O). The tool auto-detects the structure column, recognizing common header names: SMILES, smiles, SMI, and canonical_smiles. You can keep any other columns you like (names, IDs, measured activity) — they ride along untouched.

Upload it

Drag & drop the file onto the uploader (or click to browse). The tool handles files with thousands of rows. As it works, a progress bar shows how far along it is, so a large library doesn't leave you guessing.

Pick properties / LEIs and run

Choose which of the 28 molecular properties and 8 ligand-efficiency indices (LEIs) you want, then press run. Molecules are processed in parallel (several at a time) so big batches finish quickly. If a row contains an invalid or unreadable structure, that single molecule is reported as failed and skipped — the rest of the batch keeps going. One bad row never crashes the whole run.

Download the enriched file

When it's done, download the result: your original data plus every calculated column, in the same row order. Open it in Excel or pass it straight to your modeling pipeline. The structure-to-results link is preserved row-by-row, so nothing gets shuffled.

What an input file looks like

The simplest valid file is two columns: a label and a structure. Here is a tiny example with aspirin and caffeine. Notice the header row names the structure column SMILES — one of the names the tool auto-detects.

caffeine,CN1C=NC2=C1C(=O)N(C(=O)N2C)C

Tip — prepare a clean structure column

Spend a minute getting the structure column right and the whole batch runs smoothly. Use one of the recognized header names (SMILES, smiles, SMI, canonical_smiles); keep one molecule per cell with no stray spaces, quotes, or line breaks; and remove blank rows. If a SMILES came from another program, paste it as plain text so hidden formatting doesn't sneak in. A tidy structure column means fewer "failed" rows in the report.

What gets calculated

You choose the columns to add — there is no fixed list you're forced to take. The two families available are:

Molecular properties (28)

Molecular weight, LogP (greasiness), TPSA (polar surface), hydrogen-bond donors and acceptors, ring counts, rotatable bonds, QED, synthetic accessibility, and more — across 7 groups. See the full list →

Ligand-efficiency indices (8)

NSEI, NBEI, BEI, SEI, nBEI, mBEI, LEH, LEP — "binding power per unit of size." These need an activity value (pKi or pIC50) in your file. See ligand efficiency →

Reuses what you already have

If your file already contains columns like molecular weight, TPSA, or heavy-atom count, the efficiency calculations reuse them rather than recomputing — so your own values are respected and the run is faster.

How failures are handled

In a real library, a handful of rows are always imperfect — a typo in a SMILES, an empty cell, a structure the parser can't read. Batch Processing is built so these do not stop the run. Each problem row is recorded in a per-molecule error report (which row, and why it failed), then skipped. Everything else completes normally.

After the run, check the report

If some rows fail, the report tells you exactly which ones and why. Fix those structures in your source file and re-upload just to confirm — or accept the partial result if a few drop-outs don't matter for your purpose.

One file at a time

Each workspace holds a single file

Batch Processing works on one uploaded file at a time. Uploading a new file replaces the previous one in this workspace. If you have several spreadsheets, the clean approach is to combine them into one file first (same structure-column name across all rows), or process them one after another and save each enriched download as you go.

The output is built to travel

The enriched file is deliberately plain: clean numeric columns with no special formatting, merged cells, or hidden characters. That keeps it readable everywhere downstream. It opens directly in spreadsheet software and imports cleanly into the cheminformatics tools chemists commonly use next.

Excel KNIME Pipeline Pilot StarDrop

Because the structure column and every original field are preserved, you can join the results back to assay data, filter on a calculated column, or hand the file to a colleague's workflow without any reformatting.

Tip — batch first, then explore

A common pattern: run a batch to enrich your whole library, download the file, then upload that same enriched file into the Visualization gallery or GMM grouping to chart and cluster on the new columns. Batch turns raw structures into the numeric columns those analyses need.

Common questions

Which file types can I upload?

CSV (.csv) and Excel workbooks (.xlsx). Both should have a header row and one column of chemical structures, one molecule per row.

How does it know which column has my structures?

It auto-detects the structure column by recognizing common header names: SMILES, smiles, SMI, and canonical_smiles. Naming your column one of these is the most reliable way to be detected.

How many molecules can it handle?

Thousands. Molecules are processed in parallel, and a progress bar tracks the run so you can watch a large library complete.

What happens to rows with a bad structure?

They're reported individually (which row and why) and skipped. The rest of the batch finishes normally — one invalid SMILES never crashes the run.

Will my original columns survive?

Yes. The download is your original file with the calculated columns appended on the right, in the same row order. Names, IDs, and activity values are all preserved.

Can I calculate ligand-efficiency indices in batch?

Yes, but those indices need an activity value (pKi or pIC50) in your file, because efficiency means binding strength relative to size. See Ligand efficiency for what to include.
