WORKING WITH YOUR DATA

Custom calculations

Sometimes the number you want isn't one of the built-in columns — it's a combination of them: a ratio, a sum, a custom score, or a yes/no flag. Custom calculations let you build that column yourself, right inside the tool — exactly like writing a formula in a spreadsheet, but without exporting anything to Excel.

What this is for

Every dataset eventually needs a column the tool doesn't provide out of the box — perhaps "polar surface area divided by molecular weight," or a flag that marks every compound heavier than 500. Instead of downloading your data, computing the column in Excel, and uploading it again, you can do it in place.

If you've ever typed something like =A2/B2 into a spreadsheet cell, you already understand the idea. You give the new column a name and a short recipe that refers to columns you already have; the tool fills in the value for every row.

No coding Uses your existing columns Live preview before you keep it Charts & exports like any column
Where to find it

Open the ➕ Custom calculations panel — it appears wherever you have a table of data: Batch Processing, Data Visualization, and GMM Analysis. Any column you build there joins your table just like the built-in properties.

Writing a calculation

A calculation has two parts: the name of the new column, and the recipe that produces it. To use an existing column in the recipe, wrap its name in square brackets — for example [TPSA] means "the value in the TPSA column for this row." Combine columns with ordinary arithmetic: + add, − subtract, * multiply, / divide.

For example, a simple "how polar is this molecule for its size" column is polar surface area divided by molecular weight:

[TPSA] / [Molecular_Weight]

You can build longer recipes too, with parentheses to control the order — just like maths class:

([Molecular_Weight] + 10 * [TPSA]) / [HB_Acceptors]
Let the editor fill in column names

You don't have to remember exact spelling. Start typing a column name (or a function) and the editor pops up a list of matches — pick from the list and it inserts the full [Column] for you. One small habit: don't type the opening [ yourself before choosing — the suggestion already includes it, so let it do the bracket.

What you can put in a recipe

Beyond + − * / you have a small toolbox of helper functions. You'll rarely need more than the first row, but the rest are there when you do.

Safe by design

A recipe can only do maths and use the helpers above — it cannot run arbitrary programs or touch anything outside your data. It runs in a protected sandbox, so there's no way for a calculation to do something unexpected to your computer or your file.

Try it before you keep it

Nothing is added to your table until you say so. The workflow is four small steps:

Name the column

Type a name for the new column (for example, polarity_ratio). As you type, the panel highlights it among the columns you already have, so you can see exactly what you're about to create.

Write the recipe

Enter your calculation in the editor — or pick a ready-made preset from the dropdown for common calculations. The moment it's valid, you'll see a green check.

Check the preview

A small preview table shows the result on the first few rows. This is just a draft — your real data hasn't changed yet. Read it, sanity-check the numbers, and adjust the recipe if needed.

Apply it

Click Apply column and the new column is added to your table for real. The form then clears so you can build the next one, and every calculation you've applied is listed below for review.

Preview ≠ added

Seeing your column in the preview table does not mean it's in your data yet — the preview is a draft of the first rows so you can check it. The column only joins your table (and your charts, groupings, and downloads) once you click Apply column.

Building on your own columns

Calculations stack. Once you've applied one column, you can use it inside the next calculation, just like any built-in column — so you can build a result up in readable steps rather than cramming everything into one giant recipe.

For instance, apply a column called polarity_ratio first, then write a follow-up that refers to it:

polarity_ratio   =   [TPSA] / [Molecular_Weight]
flagged          =   if([polarity_ratio] > 0.5, "polar", "non-polar")
Re-using a name updates that column

If you give a new calculation the same name as an existing column, applying it overwrites that column with the new recipe — the panel tells you this before you apply, so it's never a surprise. This is how you tweak a calculation: just re-apply it under the same name.

Save and reuse your calculations

The set of calculations you've built can be exported to a small file and imported into another dataset later. That means you can define a standard set of derived columns once — a house style of ratios and flags your group always wants — and apply it to every new spreadsheet with a couple of clicks, instead of re-typing them each time.

Where your new columns go

A column you build behaves exactly like a built-in property. You can put it on a chart axis, cluster on it in GMM grouping, sort and filter on it, and it's included when you download your results. There's no second-class treatment — once applied, it's just another column in your table.

Common questions

Why single square brackets, and not double?

A column reference is wrapped in one pair of brackets: [TPSA]. Double brackets like [[TPSA]] are read as something else and won't work. The easiest way to avoid the mistake is to pick the column from the suggestion list rather than typing the brackets yourself — it inserts the correct single-bracket form.

Can I use a column I just created?

Yes. After you apply a calculation, its column is available to the next one — so you can build results up in steps. The editor will suggest it just like any other column.

How do I remove a calculation?

Every calculation you've applied is listed in the panel with a small remove button. Removing one drops its column from your table; the rest are unaffected.

What happens to rows with a missing or blank value?

If a row is missing one of the values the recipe needs, that row's result is left blank rather than guessed. The preview tells you how many of the sampled rows came out blank, so you can spot gaps in your data early.

Could a calculation harm my data or my computer?

No. Recipes are limited to maths and the listed helper functions, and run in a protected sandbox. A calculation can only produce a number (or a short text label) from your existing columns — it can't run programs or reach outside your data.
